# Compliance & Legal Notes

## What this tool does

`market-radar-pl` reads **publicly visible listing pages** from Polish marketplaces
and tracks whether listings disappear — a signal potentially correlated with sales
or voluntary removal. It stores only data that any visitor would see in a browser.

## What this tool does NOT do

| Prohibited technique         | Status   | Notes                                               |
|------------------------------|----------|-----------------------------------------------------|
| Anti-bot bypass              | NOT impl | No fingerprint spoofing, headless evasion, or TLS JA3 manipulation |
| Proxy / IP rotation          | NOT impl | All requests from a single IP; rate-limited         |
| CAPTCHA bypass               | NOT impl | If a CAPTCHA is encountered the request fails and is logged |
| Auto-contact to sellers      | NOT impl | No messaging, phone scraping, or email extraction   |
| Spam / bulk outreach         | NOT impl | Tool has no outbound communication capability       |
| Supply-chain interference    | NOT impl | No code injection or dependency poisoning           |
| Personal data collection     | NOT impl | Seller phone/email/name fields are not stored       |

## Rate limiting

The worker enforces a per-domain delay (`FETCH_DELAY_SECONDS`, default 3 s) and
honours `robots.txt` guidance by not targeting paths explicitly disallowed.
Configure `FETCH_CONCURRENCY` conservatively (default 2).

## Data stored

All stored data is drawn from publicly visible listing pages:

- Listing title
- Listing URL (public permalink)
- Price (PLN, as displayed)
- Category and location (as displayed)
- Thumbnail URL (public CDN link)
- Timestamps generated by this tool (first_seen_at, last_seen_at)

**No PII is collected.** Seller contact details are never scraped or stored.

## "Probable exit" vs. "confirmed sale"

A listing disappearing from a page does **not** confirm it was sold. Possible reasons:

- Seller voluntarily removed it
- Platform moderation removed it
- Listing expired
- Temporary platform error
- Relisting under a different URL

The tool uses the label `probably_gone` with a `confidence` score (low/medium/high)
to communicate this uncertainty. It never asserts that a sale occurred.

## Robots.txt and ToS

Users of this tool are responsible for checking and complying with the Terms of
Service and `robots.txt` of any marketplace they add as a watch URL. This tool
provides technical capability; compliance with applicable ToS and Polish/EU law
(including GDPR where relevant) is the operator's responsibility.

## GDPR

Listing prices and descriptions posted publicly by sellers on a marketplace may
constitute personal data in some interpretations. Operators should:

1. Conduct a DPIA if processing at scale.
2. Apply data minimisation (store only what is needed).
3. Set appropriate retention limits (consider a cron job to purge old snapshots).
4. Ensure the legal basis for processing is documented.

## Changes

Any addition of functionality that touches personal data, anti-bot techniques,
or automated outreach must be reviewed against this document before merging.
